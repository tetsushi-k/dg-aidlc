# 日本語ドキュメント（DG-AIDLC 上乗せ）

`aidlc-docs/` およびテンプレ由来の日本語は、**自然な日本語**で書く。機械リンタではなく、生成・編集時のハードルール。題材固有の固有名詞は例に使わない（キットはドメイン非依存）。

## 守ること

- 現場役・採用担当が音読して引っかからない文にする
- 一覧・表は短く。長い説明は本文に回す
- コード名・英語の識別子はそのままでよい。周辺の説明文は日本語として自然に

## 避けること（翻訳調・曖昧さ）

| 避ける例 | 直し方の例 |
|----------|------------|
| 断りなき〜 / 特段の記載なき場合 | 特に書かないときは〜 |
| 〜を足してよい（何を足すか不明） | その行にだけ追加で書く |
| 主に使う〜。A のときも B | 省略時は B（A を含む） |
| 基準時刻 + まだ残っている分（回りくどい時間の言い方） | 基準時刻より後（まだ有効）など、短く言う |
| please describe / Other をそのまま | その他（…に希望内容を書く） |

## チェック（ファイル書き込み前）

1. 修飾語が漢文調・翻訳調になっていないか
2. 1 セル／1 行に説明を詰め込みすぎていないか
3. 例文が特定作品のドメイン語だけになっていないか（キット・テンプレ）
4. 質問文は `.cursor/rules/dg-aidlc.mdc`「現場役が読めること」も満たすか
